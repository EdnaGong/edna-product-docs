# 515 需求文档补充说明

## 一、全局策略补充（补充至 4.2 章节开头）

### 1. 单次回答多卡片规则

- **排列顺序**：卡片紧跟对应的文字内容段落之后展示，不集中堆叠
- **数量上限**：单次回答最多展示 **3 张**卡片；超出时按触发顺序取前 3 张
- **互斥规则**：参配卡与配置图解卡不在同一回答中同时出现；车型卡与口碑卡可共存

### 2. 卡片位置统一规则

- 默认位置：对应文字内容段落结束后、下一段落开始前插入
- 新车日历：固定在段首总结之后（已有定义，保持不变）
- 配置图解卡 / 车型亮点配置图解：固定在模型文字介绍完成后插入

### 3. 接口异常降级策略（通用）

| 异常类型 | 处理策略 |
|---|---|
| 接口超时（>3s） | 不展示卡片，静默降级，不影响文字回答 |
| 接口报错 / 返回异常 | 不展示卡片，静默降级 |
| 数据为空 | 按各卡片异常策略执行（已有定义） |
| 部分字段缺失 | 按各卡片字段异常策略执行（已有定义） |

## 二、各卡片补充

### 车型卡

**补充：车型图取图规则**
- 接口返回多个年款时，取当前在售的最新年款图片
- 若最新年款无图，依次向前取有图的年款

**补充：指导价数据说明**
- 指导价为缓存数据，更新频率由数据侧定义，非实时价格
- 卡片上不展示价格时效说明

---

### 榜单卡

**补充：当月榜单未出时的处理策略**
- 当月榜单数据未就绪时，自动展示上月数据
- 卡片标题标注实际月份，如"3月销量榜"，不标注"最新"

**补充：同一回答涉及多种榜单类型**
- 优先展示与用户 query 最相关的 1 种榜单卡
- 若 query 明确涉及多种排名（如"续航和销量哪个榜好"），最多展示 2 张榜单卡

---

### 新车日历

**补充：展示时间范围**
- 展示未来 **1 个月内**即将上市的新车

**补充：排序规则修正**
- 原文"按照上市时间倒序"表述有歧义，修正为：**按上市时间升序排列**（最近要上市的排在最前）
- 有明确上市时间的排在无上市时间的前面（保持原有规则）

**补充：小角标规则**
- 小角标内容由车系接口返回，产品侧定义支持的角标类型：新车上市 / 预售中 / 即将发布
- 接口未返回角标字段时不展示

---

### 口碑卡

**补充：车系 vs 车型维度明确**
- 口碑卡为**车系维度**，展示该车系的综合口碑评分
- 触发条件中"车型口碑"的表述统一修正为"车系口碑"

**补充：维度评分展示顺序**
- 按固定顺序展示：动力 / 油耗 / 操控 / 舒适性 / 外观 / 内饰 / 空间 / 性价比
- 某维度数据为空时跳过该维度，不占位

---

### 车图卡

**补充：车系级 query 取图规则**
- 用户未指定年款时，默认取当前在售主销年款的图片
- 主销年款由数据侧定义

**补充：图片加载顺序说明**
- 按接口返回顺序展示，接口侧按图片质量/热度排序（由数据侧定义）

**补充：卡片图片数量规则明确**

| 场景 | 图片来源 | 卡片展示数量 |
|------|------|------|
| 明确年款车型 | 该车型图库，按接口返回顺序 | 最多 6 张 |
| 仅指定车系 | 该车系图库中排列第一的车型 | 最多 5 张 |

- 返回几张展示几张，不足不补位
- 第 6 张图叠加半透明蒙层，蒙层文案：「查看更多（共 XX 张）」，XX 为图库该类型实际总数
- 图库该类型图片 ≤ 5 张时，不展示「查看更多」蒙层

**补充：大图预览交互详细说明**

**进入大图**
- 点击卡片内任意一张图片，进入全屏图片预览
- 进入后默认展示用户点击的那张图片

**大图页面结构**
```
┌────────────────────────────┐
│ ← 返回按钮     3 / 6        │  ← 顶部导航栏（透明浮层）
├────────────────────────────┤
│                            │
│         全屏图片展示         │
│                            │
├────────────────────────────┤
│ [外观] [内饰] [座椅] [夜景]  │  ← 图片类型 Tab（有多类型时��示）
│  ○ ○ ● ○ ○ ○              │  ← 进度指示点
│    查看更多（共 XX 张）      │  ← 条件展示
└────────────────────────────┘
```

**图片切换**
- 切换范围：当前卡片展示的图片（明确车型最多 6 张，车系最多 5 张）
- 操作方式：左右滑动手势 / 点击图片左右半区
- 滑动到首张左滑无响应，末张右滑无响应，不循环
- 计数器实时更新，格式：`当前序号 / 可查看总数`，如 `3 / 6`

**底部图片类型 Tab**
- 仅当接口同时返回多种图片类型时展示
- 切换 Tab 后：图片列表替换为对应类型，从第 1 张开始，计数器重置

**大图内「查看更多」**
- 展示条件：图库该类型图片总数 > 当前卡片展示数量
- 按钮文案：「查看更多（共 XX 张）」
- 点击跳转：车型图库页，定位到对应图片类型；无明确分类默认定位外观

**关闭大图**
- 点击顶部返回按钮 `←`
- 向下滑动手势（下滑超过 100px 触发）
- 关闭后回到用户点击前的对话滚动位置

**异常策略**

| 场景 | 处理方式 |
|------|------|
| 整体无图数据 | 不展示车图卡 |
| 单张图加载失败 | 展示默认占位图（灰色背景 + 汽车之家 logo）|
| 图库总数为 0 | 不展示「查看更多」按钮 |
| 图片未加载完 | 展示骨架屏，加载完成后替换 |

**⚠️ 待确认：** 原需求文档"明确车型取前 6 张"与交互描述"可查看显示的 5 张图"数量不一致，当前以 6 张为准，请确认对齐。

### 参配卡

**补充：参配维度数量规则**
- 维度数量不固定，由模型根据用户 query 相关性判断输出，通常为 **6-8 个**
- 用户只问单个参数时，仅输出该参数对应维度，不强制补齐
- 无上限强制要求，以回答完整性为准

**补充：多版本处理规则**
- 车系有多个在售版本时，默认展示主销版本的参配
- 用户 query 中明确指定版本时，展示对应版本
- 主销版本由数据侧定义

---

### 配置图解卡

**补充：车型列表排序定义**
- "车型列表里前 2 个车型"指按**价格升序**排列后的前 2 个车型
- 若用户 query 中明确提及具体版本，优先使用用户指定的版本进行对比
