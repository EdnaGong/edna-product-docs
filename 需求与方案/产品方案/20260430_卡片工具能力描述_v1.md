# 卡片工具能力描述

> 智能体根据卡片能力描述自主判断是否调用，替代原有意图规则触发方式。
> 来源文档：515版本需求

---

## 车型卡

```
能力描述：展示某款具体年款车型的核心参数，包含车图、品牌、级别、能源类型、续航/油耗、指导价，
         帮助用户快速掌握这款车的基本面貌。

调用时机：
- 用户的问题以某个具体年款车型为核心，诉求是了解这款车型本身

不调用时机：
- 用户问的是车系整体，没有指定年款
- 用户只是提到某款车，但核心问题不是了解车型本身（如问保养周期、如何谈价）
- 非在售汽车（二手车、摩托车等）

前置数据条件：有该年款车型的对应数据

入参：年款车型 ID
```

## 榜单卡

```
能力描述：以排序列表形式展示同类车系在某个维度上的横向排名（最多 3 条），
         覆盖月销、续航、保值、降价、口碑、实测 7 类榜单，
         帮助用户在多辆车之间快速做出横向比较。

调用时机：
- 用户想看某个维度的排行榜或排名列表，核心诉求是多车横向对比

不调用时机：
- 用户只是问某辆车排第几——直接在回答里说即可
- 用户需要的是推荐，不是排名（推荐用选车卡）
- 有效榜单数据不足 3 条

前置数据条件：该榜单类型有至少 3 条有效数据

入参：
- 榜单类型（月销榜 / 新能源月销榜 / 续航榜 / 保值榜 / 降价榜 / 口碑榜 / 实测榜）
- 各榜单筛选条件不同，具体如下：

  月销榜 / 新能源月销榜
  - 时间范围（必填）：月度 / 3个月 / 半年度 / 近1年，格式如「2025-03」「2025-Q1」
  - 筛选条件：车型级别、能源类型、价格区间（可选）

  续航榜
  - 续航类型：纯电续航 / 插混续航 / 增程续航，未指定默认纯电续航
  - 筛选条件：车型级别（可选）

  保值榜
  - 筛选条件：车型级别（可选）

  降价榜
  - 筛选条件：车型级别（可选）

  口碑榜
  - 筛选条件：车型级别（可选）

  实测榜
  - 实测类型（根据用户关注的指标选择对应类型）：
    · 续航榜：关注实测能跑多远，测实际续航里程
    · 百公里能耗榜：关注新能源车实际用电量，测百公里电耗
    · 百公里油耗榜：关注燃油车实际油耗，测百公里油耗
    · 充电榜：关注充电速度，测 30%-80% 充电用时
    · 加速榜：关注加速性能，测 0-100km/h 加速时间
    · 制动榜：关注刹车性能，测 100-0km/h 制动距离
  - 筛选条件：车型级别（可选）
```

## 新车日历

```
能力描述：展示即将上市或近期已上市的新车列表（最多 3 条），
         帮助用户快速了解近期新车动态。

调用时机：
- 用户询问近期 / 即将 / 已上市有什么新车，关注的是「有哪些新车」

不调用时机：
- 用户询问某辆具体新车的详细参数（用车型卡）
- 有效新车数据不足 2 条

前置数据条件：有至少 2 条带上市时间的新车数据

入参：
- 时间描述：用户指定的时间，如「4月」「近期」，未指定则默认展示即将上市
- 筛选条件：车型级别、能源类型（可选）
```

## 口碑卡

```
能力描述：展示某具体车系的口碑综合评分及 8 个维度评分
         （动力 / 油耗 / 操控 / 舒适性 / 外观 / 内饰 / 空间 / 性价比），
         帮助用户快速了解真实车主的综合评价。

调用时机：
- 用户想了解某具体车系的口碑评分或综合口碑表现

不调用时机：
- 用户问的是具体优缺点或某个问题的口碑描述——文字回答即可，不需要卡片
- 用户未指向具体车系（如「哪款车口碑好」）——用榜单卡的口碑榜
- 综合评分数据为空

前置数据条件：有该车系的口碑综合评分及维度数据

入参：车系 ID
```

## 参配卡

```
能力描述：以表格形式展示某车系下各车型版本的核心参数配置（如指导价、续航、充电、功率、能耗等），
         帮助用户横向对比同一车系不同版本之间的差异。

调用时机：
- 用户询问单个车系或单个车型的参配情况，且未指定某一具体参数
- 用户想整体了解这款车有哪些版本、各版本配置如何

不调用时机：
- 用户只问某一个具体参数（如「小米SU7续航多少」——直接回答即可）
- 用户是多车对比场景（对比场景有独立卡片）
- 无对应车系参配数据

前置数据条件：有对应车系的参配数据

入参：车系 ID 或年款车型 ID
```

```
         平铺 6 张可横滑查看，帮助用户直观感受车辆外观与内饰。

调用时机：
- 用户明确表达想看图，且问题指向具体车系或年款车型

- 用户没有明确看图意图（如只是问「外观好看吗」——文字回答即可）
- 无实拍图数据

前置数据条件：有该车系 / 车型的实拍图数据

入参：
- 年款车型 ID（无论用户是否指定年款，均传入本次回答中实际介绍的那款车型 ID）
- 图片类型：外观 / 内饰 / 座椅 / 夜景，用户未指定时默认外观
```
