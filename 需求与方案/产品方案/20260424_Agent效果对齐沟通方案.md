# Agent 效果对齐沟通方案
**时间**：2026-04-24 下午  
**目的**：基于真实 bad case，对齐 Agent 回答质量问题及后续优化方向

---

## 一、问题现象（bad case 实录）

从收集的 case 中，问题集中在以下 **3 类**：

---

### 问题类型 A：回答焦点偏移，信息过载

**Case 1：高山7上市时间**

| | 内容 |
|--|------|
| 用户 query | 高山7上市时间 |
| 当前输出 | 回答了上市时间，但随即展开了完整的车型介绍（空间/动力/舒适性）和购车建议，输出超过500字 |
| 期望输出 | 直接告知上市时间，1-2句话 |
| 问题标注 | 实体识别能力 |

**差距描述**：用户只问了一个事实性问题，模型识别出车型后，套用了"场景化解读"的回答模板，答非所问，信息严重过载。

---

### 问题类型 B：时效信息处理不够直接，结构混乱

**Case 2：内蒙古买新车的置换补贴停止了吗**

| | 内容 |
|--|------|
| 用户 query | 内蒙古买新车的置换补贴停止了吗 |
| 当前输出 | 先给结论，但随后铺垫了大量风险提示，结构散乱，核心答案被稀释 |
| 期望输出 | **加粗结论放首位** → 政策时间节点 → 简要说明注意事项 |
| 问题标注 | 时效信息、引导问效果 |

**差距描述**：结论正确，但输出格式和信息层级不符合用户预期，用户需要扫读才能找到核心答案。

**Case 3：今天CRV混动的1.5t，现在什么价位？**

| | 内容 |
|--|------|
| 用户 query | 今天CRV混动的1.5t，现在什么价位？ |
| 当前输出 | 先纠偏（混动版不是1.5T），再给价格，结构合理，但表达方式偏罗列 |
| 期望输出 | 纠偏加粗突出 → 价格信息清晰呈现 |
| 问题标注 | 时效、纠偏 |

---

### 问题类型 C：未直接正面回答用户问题

**Case 4：零跑c01增程版怎么样 增程器噪音大吗 电池质量如何**

| | 内容 |
|--|------|
| 用户 query | 零跑c01增程版怎么样 增程器噪音大吗 电池质量如何 |
| 当前输出 | 开篇用了大量总结性评价铺垫（"堆料很足、性价比极高"），再进入分点回答；用户的三个具体问题被稀释在评价语境中 |
| 期望输出 | 三个问题各自有明确结论（噪音：日常正常/高速偶有、电池：口碑较好/质保X年），再给综合评价 |
| 问题标注 | 未直接正面回答用户问题、引导问效果 |

**差距描述**：用户问了三个明确的具体问题，模型先给"综合评价"再分点，导致用户要读完整段才能找到自己想要的答案。

---

### 问题规律汇总

| 问题类型 | 涉及 case 数 | 核心表现 |
|----------|-------------|----------|
| 回答焦点偏移，信息过载 | 2条 | 用户问A，模型答A+B+C+D |
| 输出格式/结构不符合预期 | 3条 | 结论不突出，层级混乱，缺乏 markdown 结构 |
| 未直接正面回答 | 2条 | 先铺垫再答，用户找核心答案成本高 |

## 二、根因分析（产品视角）

### 根因1：回答策略与 query 类型不匹配

当前模型对大多数 query 执行的是**「场景化解读模板」**——这套模板适合用户主动问"这款车怎么样"，但不适合：
- **事实性查询**（上市时间、价格、参数）→ 应直接给答案，不展开
- **多点问题**（噪音+电池+口感）→ 应逐点直接回答，不先做综合评价
- **时效/政策查询** → 应结论前置，不先铺垫背景

本质是：**模型缺少对 query 信息需求类型的区分能力**，用一种回答模板覆盖了所有场景。

### 根因2：输出格式规范未被模型执行

从 case 对比来看，期望输出普遍有：
- 关键结论加粗
- 分点用 `###` 标题而非纯文字
- 分割线区隔不同问题

当前输出缺乏这些格式规范，说明**模型的输出格式 prompt 约束不够，或未被稳定执行**。

### 根因3：引导追问时机不准确

部分 case（内蒙古补贴、高山7）在核心问题还没答清楚的情况下就开始引导用户说明场景。**引导追问应在回答完用户问题之后，而不是替代回答。**
