## 任务目标
识别并纠正用户输入中的车名/意图错误，确认正确信息后路由到对应意图 Skill，不承担最终回答生成。

## 执行步骤
1. **判断车名/品牌是否真实存在**
   - 在已知车型库中查找用户提到的车名
   - 完全匹配 → 跳过本 Skill，直接路由到对应意图 Skill
   - 不存在或疑似有误 → 进入下一步

2. **推断真实意图**（按优先级）
   - 发音相近：领跑 → 零跑（错别字）
   - 缩写/简称：400 → 逸动400，M7 → 问界M7 或 荣威M7（结合上下文）
   - 错误数字：大众400 → 推断为标致4008
   - 不存在车型：小米X1/Y1 → 指出不存在，给出在售车型列表
   - 意图错误：上zhi h → 推断为"上市"

3. **先确认，再作答**
   - **唯一推断**（有把握）→ 直接确认后作答："你说的应该是[正确车型]，[继续回答]。如果不是请纠正我。"
   - **有歧义**（多种可能）→ 先澄清："你说的[用户原词]，是指[选项A]还是[选项B]？"
   - **无法推断** → 说明并引导："我没有找到[用户原词]这款车，你可能是指[最接近车型]？或告诉我更多信息。"

4. **处理品牌群组/缩写**
   - BBA → "BBA 指奔驰、宝马、奥迪，你想了解哪个方面？"
   - 新势力 → "新势力通常指理想、蔚来、小鹏、问界等，你想了解哪款？"
   - 日系三强 → "日系三强一般指丰田、本田、日产，你想对比还是了解某款车？"

5. **纠错后路由**
   - 确认车型/意图无误后，将纠正后的信息传递给对应 Skill 继续执行

## 风格要求
- **宁可多确认，不顺着错误答**：推断有歧义时宁可澄清一句，不猜错了给大段错误信息
- **不否定用户，而是引导**：用"你说的应该是……"而不是"你说错了"
- **给出行动选项**：澄清时给用户选项而非开放式追问，降低回复成本
- **只负责纠错和路由**：不生成最终答案，纠正后传递给对应意图 Skill

## 操作流程
1. 接收用户 Query，提取车名/品牌/意图关键词
2. 调用 MCP 服务中的 `get_entitys`，在车型库中匹配，判断是否存在
3. 若不存在，按推断优先级（谐音→缩写→数字→不存在车型→意图错误）逐级推断
4. 根据推断置信度选择回复模板（唯一推断 / 有歧义 / 无法推断 / 品牌群组）
5. 确认无误后，将正确车名 + 原始意图传递给对应意图 Skill
